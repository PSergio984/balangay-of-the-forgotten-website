# Phase 2 Plan: Wiki Core & Content Engine

## Goal
Establish a professional headless CMS (Payload 3.0) to manage game lore, character stats, and news bulletins, while rendering them in an immersive pixel-art wiki.

## Success Criteria
1. Payload CMS 3.0 integrated with Next.js 15 and Cloud PostgreSQL.
2. Data models implemented for Characters, Bosses, Minibosses, Relics, and Locations.
3. Immersive 'Ancient Scroll' UI wrapper created for all wiki entries.
4. Dynamic routing established at `/wiki/[category]/[slug]`.
5. Bulletin Board on landing page is data-driven from the CMS.

## Tasks

### 1. Infrastructure Setup
- [ ] Install Payload 3.0 dependencies (`payload`, `@payloadcms/next`, `@payloadcms/db-postgres`, etc.).
- [ ] Configure `next.config.ts` with `withPayload`.
- [ ] Initialize `payload.config.ts`.
- [ ] Setup `.env` with `DATABASE_URL` and `PAYLOAD_SECRET`.

### 2. Data Modeling (Schema Implementation)
- [ ] Implement `Characters` collection (HP, ATK, MAG, DEF, Role Presets).
- [ ] Implement `Bosses` collection (HP, ATK, MAG, DEF, Movesets, Locations).
- [ ] Implement `Minibosses` collection (Linked to Bosses/Locations).
- [ ] Implement `Relics` collection (Linked to Bosses/Locations).
- [ ] Implement `News` collection (For Bulletin Board).
- [ ] Implement `Locations` collection (Hierarchical).

### 3. Frontend Implementation
- [ ] Create `AncientScrollContainer` wiki variant.
- [ ] Implement dynamic routes at `app/wiki/[category]/[slug]/page.tsx`.
- [ ] Build "Stat Block" UI components for HP/ATK/DEF/MAG display.
- [ ] Implement "See Also" bi-directional links using Payload `join` fields.
- [ ] Update `BulletinBoard.tsx` to fetch data from Payload.

### 4. Search & Discovery
- [ ] Implement global search component (WIKI-02).
- [ ] Configure `generateStaticParams` for SEO and performance.

## Verification Strategy
- **CMS Test**: Create an admin user and verify entry creation in all collections.
- **Data Test**: Verify relational links (e.g., clicking a Boss shows their Relics).
- **UI Test**: Ensure "Ancient Scroll" wrapper renders correctly on mobile/desktop.
- **Performance Test**: Check that `generateStaticParams` builds static pages for all wiki entries.
