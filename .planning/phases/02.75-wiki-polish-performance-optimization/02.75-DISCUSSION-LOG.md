## Image loading on deployment

| Option | Description | Selected |
|--------|-------------|----------|
| Direct payload uploads | Store uploads in default `/media` folder at root, which is gitignored and not deployed. | |
| Local/public media sync | Save media uploads to `public/media` folder and commit these assets to Git so they deploy statically. | ✓ |

**User's choice:** Local/public media sync
**Notes:** The user agreed to proceed with saving to `public/media/` and committing seeded media files to Git so they load correctly on serverless platforms.

---

## Page loading performance

| Option | Description | Selected |
|--------|-------------|----------|
| Dynamic Server-side rendering | Query Payload CMS/database on every request. | |
| Static Site Generation (SSG) with ISR | Pre-generate all wiki and category dynamic pages at build time using `generateStaticParams` and cache with ISR. | ✓ |

**User's choice:** Static Site Generation (SSG) with ISR
**Notes:** Pre-generating dynamic pages at build time guarantees instantaneous load times when users navigate.

---

## Sticky Footer shared layout

| Option | Description | Selected |
|--------|-------------|----------|
| Inline Homepage Footer | Keep the footer on the home page only, leaving it floating or missing on other pages. | |
| Layout Sticky Footer | Move footer to a shared component in `layout.tsx` and structure with flexbox sticky rules. | ✓ |

**User's choice:** Layout Sticky Footer
**Notes:** Shared sticky layout structure forces the footer to remain at the bottom of the viewport even on pages with short content.
